# 产品首页详细计划

## 定位

首页是 AI RSS Web 的产品入口，用来展示项目能力、支持渠道和整体数据流。

它不是普通说明文档页，也不是简单导航页，而应该让用户一眼明白：

- AI RSS Web 能把 RSS 信息源转成更易读的 AI 摘要新闻流。
- 它支持 Web、Mobile App、CLI、VS Code / Cursor 插件、Chrome 插件多个展示渠道。
- 后台负责生产数据，所有前端形态都是只读展示端。
- 用户可以从首页进入 Web 阅读器，并了解未来其他端的能力。

## 首页目标

必达目标：

- 好看、有产品感。
- 第一屏明确传达 AI RSS Web 的核心能力。
- 清晰展示支持渠道。
- 解释 backend 到多端展示的数据流。
- 提供进入 Web 阅读器的主入口。
- 为后续 mobile、cli、ide-extension、browser-extension 留出下载或文档入口。

不做：

- 不在首页塞后台管理功能。
- 不把首页做成纯文档目录。
- 不把多端渠道隐藏在普通文字段落里。

## 信息架构

建议首页结构：

```text
Hero
  -> 产品名、核心描述、进入阅读器 CTA

Capability Strip
  -> RSS 聚合、AI 摘要、评分分类、只读多端、静态 JSON / Supabase

Channels
  -> Web、Mobile App、CLI、VS Code / Cursor、Chrome

How It Works
  -> RSS 源 -> backend 生产 -> 数据层 -> 多端展示

Preview
  -> 今日精选 / 分类新闻 / 终端输出 / 插件侧边栏的组合预览

Roadmap
  -> 当前可用、计划中、后续增强

Footer
  -> GitHub、Docs、进入 Web 阅读器
```

## 第一屏 Hero

第一屏需要直接出现产品名和核心能力。

建议文案方向：

```text
AI RSS Web
把信息源整理成每天可读的 AI 新闻简报
```

辅助文案：

```text
RSS 抓取、AI 摘要、评分分类和多端只读展示。你可以在 Web、App、CLI、编辑器和浏览器插件里阅读同一批新闻。
```

主操作：

- 进入 Web 阅读器

次操作：

- 查看多端计划
- 查看数据流

## 支持渠道展示

首页必须明确展示五个渠道：

### Web

定位：

- 浏览器中的完整阅读体验。

展示重点：

- 今日精选。
- 分类看板。
- 新闻卡片。
- 主题切换。

状态：

- 当前核心端。

### Mobile App

定位：

- 真正移动端风格的新闻阅读 App。

展示重点：

- 底部 Tab。
- 下拉刷新。
- 详情页。
- 本地缓存。

状态：

- 规划中或开发中。

### CLI Reader

定位：

- 终端里的快速新闻阅读器。

展示重点：

- `ai-rss today`
- `ai-rss search openai`
- JSON 输出。

状态：

- 规划中或开发中。

### VS Code / Cursor 插件

定位：

- 编辑器侧边栏里的 AI RSS 阅读器。

展示重点：

- Activity Bar。
- Today 侧边栏。
- 搜索和打开原文。

状态：

- 规划中。

### Chrome 浏览器插件

定位：

- 浏览器弹窗、侧边栏或新标签页中的轻量阅读入口。

展示重点：

- Popup 今日精选。
- 分类切换。
- 浏览器内快速打开原文。

状态：

- 规划中。

## 能力展示区

建议展示五类能力：

- RSS 聚合：从多个来源收集信息。
- AI 摘要：把长内容压缩成可读摘要。
- 评分分类：帮助快速筛选值得看的内容。
- 双数据源：支持 static JSON 和 Supabase。
- 多端只读：同一批数据在多个端展示。

这些能力应该用短句和视觉卡片展示，不要写成长段说明。

## 数据流展示区

需要让用户理解职责边界：

```text
RSS Sources
  -> Backend Jobs
  -> Supabase / Static JSON
  -> Shared News Client
  -> Web / App / CLI / IDE / Chrome
```

视觉上可以做成横向流程、轨道图或节点图。

重点文案：

```text
后台生产数据，多端只读展示。
```

## 预览展示区

建议做成多端预览组合，而不是单张 Web 截图：

- Web 看板预览。
- Mobile 手机框预览。
- CLI 终端输出预览。
- VS Code 侧边栏预览。
- Chrome Popup 预览。

第一版没有真实端时，可以先用静态 mock 预览，后续替换为真实截图。

## 视觉方向

建议方向：信息工作台 + 新闻编辑室。

关键词：

- 清晰。
- 可信。
- 有一点技术感。
- 不做常见紫色渐变 AI 首页。
- 不做纯营销页。

色彩建议：

- 背景使用浅色或深浅分区。
- 主色可以使用墨绿、钴蓝、石墨黑之一。
- 使用一个暖色强调新闻和重要信息。
- 避免单一蓝紫渐变。

组件建议：

- 渠道卡片。
- 状态标签。
- 数据流节点。
- 真实或 mock 的产品预览。
- CTA 按钮。

## 路由建议

当前 Web 阅读器可以保留为主应用页面，首页作为新的入口。

建议：

```text
/           -> 产品首页
/reader     -> Web 阅读器
/admin      -> 管理页面，可选
```

如果当前 `/` 已经是阅读器，可以先用：

```text
/home       -> 产品首页
/           -> Web 阅读器
```

但长期建议让 `/` 成为首页，让 `/reader` 承载阅读器。

## 第一版功能清单

必做：

- 首页路由。
- Hero。
- 支持渠道展示。
- 能力展示。
- 数据流展示。
- 多端预览区。
- 进入 Web 阅读器按钮。
- 响应式布局。

可后置：

- 真实 App / CLI / 插件截图。
- 下载入口。
- 发布状态自动展示。
- 动态演示。
- 公开文档入口。

## 验收标准

- 首页第一屏明确出现 AI RSS Web 和核心能力。
- 首页明确展示 Web、Mobile App、CLI、VS Code / Cursor 插件、Chrome 插件。
- 首页能解释 backend 生产数据、多端只读展示的边界。
- 首页有进入 Web 阅读器的入口。
- 首页视觉上像产品入口，不像 README 页面。
- 移动端和桌面端布局都不拥挤、不重叠。
